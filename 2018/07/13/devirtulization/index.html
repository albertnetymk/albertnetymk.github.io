Devirtulization in C++

Jul 13 2018

Came across this commit while working with OpenJDK. The whole stuff was beyond me at first sight, so here’s a simplified version to capture the idiom.

Using the #if switch, we can control the devirtulation.

Comparing slow.s with fast.s, we can see the key difference lies in the body of myfunc1. In the fast case, puts is inlined, hence a direct call, in contrast to the indirect call in the slow case.

On second thought: this optimization assumes that the concrete type has no override in subtypes; otherwise, the use of virtual call is necessary.

If we mark the concrete type as final, compiler can perform this optimization automatically.
